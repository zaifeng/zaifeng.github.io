Go Workspace初试

2023年5月24日
 | golang
 | 阅读

​ 为了解决多个有依赖的Module本地同时开发的问题，Go 1.18引入了工作区模式。您可以将其视为一个项目文件夹，其中包含您的Go应用程序和依赖项。在Go Workspace中，您可以创建自己的包和库，或者使用其他人创建的包和库。通过设置Go Workspace，可以轻松地分享Go应用程序和依赖项，并使用Go命令来构建、测试和运行应用程序。

下面我们来简单体验一下，如何使用workspace

  1. 创建目录hiworkspace
1
2

$ cd $GOPATH/src
$ mkdir hiworkspace
  1. 初始化工作区
1

$ go work init
  1. 添加依赖库
1
2
3

$ mkdir tools
$ cd tools
$ go mod init github.com/zaifeng/tools

注意：这里创建目录并使用go mod初始化，添加其他依赖库 可以使用git clone 指定或默认目录即可

github.com/zaifeng/tools 为你的代码包里的依赖

之后，如果我们想修改依赖库代码，直接修改本地目录文件即可，测试通过后提交代码即可，并不会对上线造成任何影响。

最后注意，不要把go.work加入版本控制库，会导致编译失败 在.gitignore文件中加入即可

总结

使用workspace管理所有文件和依赖项的主要优点是：

  1. 使得依赖项的管理更加轻松和可预测。
  2. 便于协作开发。
  3. 可以方便地在不同的机器上重新创建相同的开发环境。
